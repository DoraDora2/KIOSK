## Problem Statement
식당 영업 시작 단계에서 가장 먼저 필요한 것은 손님이 헤매지 않고 바로 주문할 수 있는 간단한 주문 화면이다.
외주 키오스크를 붙이면 빠르게 시작할 수는 있지만, 메뉴 변경 속도, 화면 구성, 추가 기능 우선순위가 매장 운영 현실과 자주 어긋난다.

이 프로젝트의 첫 버전은 "한 매장의 실제 주문을 바로 받을 수 있는 웹 키오스크"다.
고급 기능보다 먼저 해결해야 하는 것은 세 가지다.

1. 손님이 1분 안에 메뉴를 찾고 담을 수 있어야 한다.
2. 총액 계산이 틀리지 않아야 한다.
3. 점주가 메뉴 구조를 코드에서 쉽게 관리할 수 있어야 한다.

## Demand Evidence
- 사용자는 "식당 영업을 시작하려고 해"라고 말했다. 이건 가벼운 아이디어가 아니라 실제 운영 목적이다.
- 사용자는 "키오스크를 외주 맡기는 것 보다는 키오스크 서비스를 직접 만드는게 좋을것 같아서"라고 말했다. 비용 절감만이 아니라 운영 통제권을 직접 가져오려는 의도가 분명하다.
- 메뉴도 이미 정식, 곁들임, 음료로 나눠서 가격까지 확정되어 있다. 첫 릴리스에 필요한 입력값이 이미 준비돼 있다는 뜻이다.

## Status Quo
지금 가능한 선택지는 보통 세 가지다.

1. 외주 키오스크를 도입한다.
2. 직원이 직접 주문을 받는다.
3. 종이 메뉴나 간단한 POS로 버틴다.

문제는 셋 다 초반 운영에 불편이 있다.
외주는 수정이 느리고, 직원 주문은 피크 타임 병목이 생기고, 종이 메뉴는 주문 누락과 합산 실수가 생긴다.

## Target User & Narrowest Wedge
첫 타깃 사용자는 매장 입구 또는 카운터 앞에서 직접 주문하는 손님이다.

- 한국어로 주문하는 일반 손님
- 메뉴를 빠르게 훑고 바로 담고 싶어 하는 사람
- 처음 방문해도 설명 없이 사용할 수 있어야 하는 사람

가장 좁고 강한 첫 버전은 이렇다.

- 한 매장 전용 웹 키오스크
- 로그인 없음
- 메뉴 카테고리 3개: 정식, 곁들임, 음료
- 터치 한 번으로 메뉴 담기
- 장바구니에서 수량 변경 및 삭제
- 총액 확인
- 결제 완료 화면과 주문번호 생성

## Constraints
- 키오스크는 터치 중심이라 버튼과 글자가 커야 한다.
- 주문 중 손님이 길을 잃지 않게 장바구니와 총액이 항상 보여야 한다.
- 메뉴 데이터는 코드 한 곳에서 관리돼야 한다.
- 첫 버전은 실제 PG 연동 없이 결제 시뮬레이션으로 끝낸다.
- 첫 버전은 한글 단일 언어, 단일 매장, 단일 메뉴판 기준으로 간다.

## Premises
1. 첫 릴리스에서는 "메뉴 탐색 속도"가 "브랜드 연출"보다 중요하다.
2. 식당 키오스크는 검색보다 카테고리 탐색이 더 빠르다.
3. 장바구니를 화면 오른쪽에 고정하면 주문 실수를 줄일 수 있다.
4. 주문 완료 경험은 실제 결제 연동 전에도 먼저 검증할 수 있다.
5. 메뉴 구조와 가격은 코드에서 분리된 데이터로 관리해야 이후 확장이 쉽다.

## Approaches Considered
### Approach A: Static HTML Single Page
정적 HTML, CSS, JavaScript 한 세트로 만든다.
배포는 쉽고 가볍다.
하지만 상태 관리가 커지면 메뉴 추가, 주문 완료 화면, 추후 결제 연동 시 유지보수가 금방 지저분해진다.

### Approach B: React + TypeScript Kiosk SPA
컴포넌트 단위로 메뉴 목록, 장바구니, 결제 다이얼로그, 완료 화면을 나눈다.
초기 세팅은 조금 더 들지만, 메뉴 변경과 기능 추가가 훨씬 쉽다.
실제 매장 운영 중 요구사항이 바뀌기 쉬운 프로젝트라 이쪽이 맞다.

## Recommended Approach
Approach B를 선택한다.

이유는 간단하다.
식당 키오스크는 처음엔 단순해 보여도 금방 "품절", "추천", "포장/매장", "실제 카드 결제", "주문 이력", "주방 디스플레이 연동"이 따라온다.
그때 정적 스크립트 뭉치보다 컴포넌트 구조가 훨씬 덜 아프다.

첫 구현 범위는:

1. 카테고리별 메뉴 보기
2. 메뉴 담기
3. 장바구니 수량 수정/삭제
4. 결제 방법 선택 UI
5. 주문 완료 및 주문번호 표시

## Open Questions

- 실제 카드 결제 단말 또는 PG와 어떤 방식으로 붙일지
- 주문 완료 후 주방으로 어떤 신호를 보낼지
- 품절 메뉴를 점주가 현장에서 어떻게 토글할지

## Success Criteria
- 손님이 메뉴를 담고 주문 완료까지 끊김 없이 진행할 수 있다.
- 모든 가격 합산이 정확하다.
- 첫 화면에서 어떤 메뉴를 눌러야 하는지 바로 이해된다.
- 데스크톱 가로 화면과 태블릿 세로 화면에서 모두 읽기 쉽다.
- 메뉴 가격과 이름을 한 파일에서 수정할 수 있다.

## Distribution Plan
첫 버전은 정적 웹 앱으로 배포한다.
운영 방식은 두 가지 중 하나다.

1. 매장 키오스크 PC 또는 태블릿에서 브라우저 전체화면으로 실행
2. 정적 호스팅에 배포 후 매장 기기에서 접속

실제 운영 시 자동 배포가 필요하면 GitHub Actions + 정적 호스팅으로 확장하면 된다.

## Dependencies
- Node.js 기반 프런트엔드 빌드 환경
- 향후 필요: 카드 결제 연동, 주문 프린터 또는 주방 디스플레이 연동

## The Assignment
실제 손님 한 명 또는 지인 한 명을 세워두고 설명 없이 주문하게 해보자.
어디에서 멈추는지 3분만 관찰하면, 우리가 화면을 고쳐야 할 지점이 바로 보인다.
